2. DATABASE FALLBACK STRATEGY
=============================

Goal:

Application should run locally without requiring PostgreSQL.

Requirements:

Primary Database

* PostgreSQL

Fallback Database

* SQLite

Behavior:

If PostgreSQL configuration is present:

Use PostgreSQL.

Otherwise:

Automatically fallback to SQLite.

Implement:

application-postgres.yml

application-sqlite.yml

Provide profile switching.

Examples:

SPRING_PROFILES_ACTIVE=postgres

SPRING_PROFILES_ACTIVE=sqlite

Default local startup should use SQLite.

Railway deployment should use PostgreSQL.

Add migrations support.

Preferred:

Flyway

Database schema must remain compatible across both engines.
