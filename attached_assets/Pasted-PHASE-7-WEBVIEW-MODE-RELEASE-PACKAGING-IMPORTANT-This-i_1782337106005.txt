PHASE 7 — WEBVIEW MODE & RELEASE PACKAGING

IMPORTANT

This is the final planned feature before release hardening.

Do NOT redesign Studio.

Do NOT add new diff rules.

Do NOT modify governance logic.

Do NOT implement SaaS features.

Do NOT introduce mock data.

The goal is to transform SpecGuard Studio into a reusable local WebView that ships with the engine.

==================================================
PRIMARY GOAL
============

Implement:

npx specguard old.yaml new.yaml --webview

Expected experience:

1. Engine analyzes contracts
2. ContractDiffReport is generated
3. Local server starts
4. Browser opens automatically
5. User interacts with full SpecGuard Studio

==================================================
ARCHITECTURE (MANDATORY)

Keep source separation.

Current structure:

root/

engine/

frontend/

Do NOT move React source into engine.

React source remains entirely inside:

frontend/

During release packaging only:

frontend/dist

must be copied into:

engine/assets/webview/

The engine should never depend on React source.

Only on compiled static assets.

==================================================

Build Engine

Build Frontend

↓

↓
